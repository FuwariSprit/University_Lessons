ゲームとか作りたい...作りたくない？

情報工学科に進んだ人間の中には、ゲームを作りたいと思っている人は少なくないはず。
そんな方々のために(自分も含めて)様々なゲームエンジンやライブラリを何番煎じだか分からないがまとめてみた。
まずライブラリやエンジンってなんぞやって人は調べてどうぞ。

っていうのは少し無責任なので、自分なりにまとめてみた。

"ライブラリ"ってなんや

プログラムに慣れ親しんでる人には馴染みがある言葉。
ゲームに限らず、何度も使うプログラムっていうのはその都度書いているとめんどくさい。(修正がめんどくさいとかあるけど、全部書くと長くなるので割愛。)
そこで関数単位でまとめたのが"ライブラリ"。
みんなが良く使うライブラリは、C言語の標準ライブラリの"stdio.h"だと思う。

そしてゲームに良く使われる処理をまとめたのが"ゲームライブラリ"ということになる。

"ゲームエンジン"ってかっこいいね(小並感)

先にいうと結構ゲームエンジンって括りは曖昧。
あとで説明がある"Unity"や"UnrealEngine"とかガッツリしたものから、"RPGツクール"や"WOLFRPGエディター"みたいなプログラムを知らなくてもゲームが作れるもの。
さらにはADVゲーム御用達の"吉里吉里"や"NScripter"までゲームエンジンと呼べる。

定義から説明するととても長くなるのでまたもや割愛。どんなことができるのかだけ簡単に書く

3Dか2Dが得意かはエンジンによるが、ある程度簡単にグラフィックの描画が可能である。中には視覚的に配置することができるものもある。
