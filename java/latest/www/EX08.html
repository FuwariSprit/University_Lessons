ゲームとか作りたい...作りたくない？

情報工学科に進んだ人間の中には、ゲームを作りたいと思っている人は少なくないはず。
そんな方々のために(自分も含めて)様々なゲームエンジンやライブラリを何番煎じだか分からないがまとめてみた。
まずライブラリやエンジンってなんぞやって人は調べてどうぞ。

っていうのは少し無責任なので、自分なりにまとめてみた。

"ライブラリ"ってなんや

プログラムに慣れ親しんでる人には馴染みがある言葉。
ゲームに限らず、何度も使うプログラムっていうのはその都度書いているとめんどくさい。(修正がめんどくさいとかあるけど、全部書くと長くなるので割愛。)
そこで関数や処理単位でまとめたのが"ライブラリ"。
みんなが良く使うライブラリは、C言語の標準ライブラリの"stdio.h"だと思う。

そしてゲームに良く使われる処理をまとめたのが"ゲームライブラリ"ということになる。

"ゲームエンジン"ってかっこいいね(小並感)

先にいうと結構ゲームエンジンって括りはかなり曖昧。
あとで説明がある"Unity"や"UnrealEngine"とかガッツリしたものから、"RPGツクール"や"WOLFRPGエディター"みたいなプログラム言語を知らなくてもゲームが作れるもの。
さらにはADVゲーム御用達の"吉里吉里"や"NScripter"までゲームエンジンと呼べる。

定義から説明すると、とても長くなるのでまたもや割愛。どんなことができるのかだけ簡単に書く

  • 3Dや2Dのグラフィックの描画 ※おまけに簡単に
  • サウンドの再生 ※おまけに(ry
  • UI関係の処理 ※おま(ry
  • データの種類ごとの管理など ※(ry

要約するとゲーム製作に関する事が便利にかつ、簡単にできるようになっている。

そして最近の3Dエンジンに多いのが、"物理エンジン"が搭載されているということだろう。
これによって、本来ならプログラム中で自力で物理演算の処理などを書かなければいけないものを、全てこの"物理エンジン"が肩代わりしてくれるのだ。

簡単とまではいかないが、本当に"ゲーム"のみに注力したいのであれば、全然アリだと思う。
実際に主は高校時代にゲームを作ったのだが、それはゲームエンジンを使って作った。
それでもかなり苦労したけれども...
